# Glossary

## Sonar / Shipwreck Terms

**SSS, Side-Scan Sonar**

A sonar sensor that creates 2D grayscale images of the seafloor. Useful for seeing object shape, texture, and acoustic shadow.

**Shipwreck**

A sunken or remaining structure of a ship on the seabed.

**Underwater cultural heritage**

수중문화유산. Includes shipwrecks, pottery, artifacts, old wooden vessels, and underwater archaeological sites.

**Anomaly**

Something that looks different from the normal seabed. It may be a shipwreck, rock, fishing gear, or noise.

**ROI, Region of Interest**

A suspicious area that should be checked again.

**Acoustic shadow**

A dark region in side-scan sonar image created because an object blocks the sonar signal. It is useful for detection but should not be labeled as shipwreck body.

## AI Terms

**Image**

The sonar-like grayscale input.

**Mask**

Pixel-level label. In this repo:

```text
white = shipwreck body
black = background
```

**Segmentation**

AI task where the model predicts which pixels belong to the target object.

**Ground truth**

The correct label or mask.

**Prediction**

The model output.

**U-Net**

A common segmentation model. It takes an image and predicts a mask.

**Dice score**

Overlap score between predicted mask and true mask.

```text
Dice = 2 * overlap / (prediction area + ground-truth area)
```

Higher is better.

**IoU, Intersection over Union**

Another overlap score.

```text
IoU = overlap / union
```

IoU is usually stricter than Dice.

**Precision**

Among predicted shipwreck pixels, how many are actually correct.

**Recall**

Among real shipwreck pixels, how many the model found.

## Simulator Terms

**Toy 2D simulator**

Simple Python image generator. It creates fake sonar-like images and automatic masks. It is not physically accurate.

**Synthetic data**

Fake data generated by code or simulator.

**Real data**

Actual sonar survey data.

**AI4Shipwrecks**

Real side-scan sonar shipwreck dataset and benchmark. Used later as the real-data baseline.

**Sim2Real gap**

Difference between synthetic simulation data and real sonar data.

**Domain randomization**

Making simulation messy and varied, such as adding noise, blur, weak contrast, and false objects, so the model does not only learn clean toy images.
